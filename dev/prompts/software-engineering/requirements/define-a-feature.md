# Define a Feature

Use this to turn a feature idea into a bounded, testable requirement.

## Prompt

Define the following feature: `[feature idea]`.

Product or system: `[context]`

Target users: `[users]`

Problem to solve: `[problem]`

Desired outcome: `[outcome]`

Known constraints: `[technical, policy, time, or compatibility limits]`

Separate the user need from the proposed solution. Produce a concise problem
statement, primary user flow, functional requirements, non-functional
requirements, edge cases, explicit non-goals, dependencies, open questions, and
testable acceptance criteria. Flag assumptions rather than silently resolving
them.

# Plan an Implementation

Use this to plan a change in the context of a real codebase.

## Prompt

Create an implementation plan for `[feature, fix, or change]` in `[repository or
project]`.

Goal: `[desired behavior]`

Constraints: `[compatibility, scope, deadlines, or technical limits]`

Relevant starting points: `[files, modules, issues, or documentation]`

Required validation: `[tests, checks, or manual flows]`

Inspect the repository and its local instructions before proposing changes.
Explain the current behavior, identify the smallest coherent change, list
affected files and interfaces, account for data or compatibility concerns, and
sequence implementation with verification. Mark unknowns that require
investigation. Do not edit files; return the plan only.

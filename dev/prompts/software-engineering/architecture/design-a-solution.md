# Design a Software Solution

Use this to explore architecture before implementation.

## Prompt

Design a solution for `[problem or capability]`.

Existing system: `[architecture and constraints]`

Users and scale: `[usage pattern, load, or growth expectations]`

Quality priorities: `[reliability, security, performance, maintainability, cost]`

Constraints: `[platform, compatibility, team, budget, or deadline]`

Describe the proposed components, responsibilities, data flow, interfaces,
trust boundaries, failure behavior, and operational needs. Compare credible
alternatives and explain the important trade-offs. Identify assumptions,
delivery risks, migration needs, and a thin first version that can validate the
design.

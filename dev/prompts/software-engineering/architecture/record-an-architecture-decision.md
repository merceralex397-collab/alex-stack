# Record an Architecture Decision

Use this to capture why a consequential technical choice was made.

## Prompt

Write an architecture decision record for `[decision]`.

Context and problem: `[background]`

Decision drivers: `[requirements and constraints]`

Options considered: `[options]`

Chosen option: `[choice, if already made]`

Evidence: `[experiments, benchmarks, or references]`

Use the sections **Status**, **Context**, **Decision Drivers**, **Options**,
**Decision**, **Consequences**, **Risks**, and **Follow-up**. Explain trade-offs
fairly, include negative consequences, and distinguish measured evidence from
assumptions.

# Prompt Library

Reusable prompts organized first by broad topic, then by the kind of work being
done. Choose the closest prompt, replace the text in `[brackets]`, remove
anything that does not apply, and attach or paste relevant source material.

## Topics

### `general-work`

- `decision-making` — compare options and challenge assumptions.
- `planning` — turn goals into plans, priorities, and schedules.
- `problem-solving` — diagnose blockers and identify useful next actions.
- `review` — critique work against an explicit purpose and criteria.

### `knowledge-and-learning`

- `research` — conduct a structured, evidence-aware investigation.
- `learning` — build understanding through explanation and practice.
- `explanation` — make a complex idea understandable to a specific audience.

### `writing-and-communication`

- `drafting` — turn notes into a coherent first draft.
- `editing` — adapt existing writing for an audience.
- `summarization` — extract the information that matters.
- `messages` — write concise email, chat, or announcements.
- `conversations` — prepare for important or sensitive discussions.
- `feedback` — give specific, fair, and actionable feedback.

### `creative-work`

- `ideation` — generate and compare varied possibilities.
- `concept-development` — turn a promising seed into a developed concept.

### `ai-and-prompts`

- `prompt-design` — improve task prompts.
- `task-briefing` — give an AI assistant a clear, bounded assignment.
- `response-evaluation` — assess an AI response before relying on it.

### `software-engineering`

- `requirements` — define a feature and its acceptance criteria.
- `planning` — plan an implementation in the context of an existing codebase.
- `architecture` — design a solution and document architectural decisions.
- `implementation` — implement or refactor code safely.
- `debugging` — investigate defects using evidence.
- `testing` — design useful tests and quality checks.
- `code-review` — review code or a pull request for consequential issues.
- `documentation` — write maintainable technical documentation.
- `security` — model threats and review security-sensitive changes.
- `performance` — investigate a performance problem with measurements.
- `maintenance` — assess and prioritize technical debt.
- `delivery-and-operations` — prepare releases and respond to incidents.

## A useful request pattern

Most prompts improve when they include:

1. **Context** — what is happening and why it matters.
2. **Goal** — the result you want.
3. **Constraints** — limits, requirements, and things to avoid.
4. **Audience** — who will use or read the result.
5. **Output** — the format and level of detail you need.

Treat generated answers as working material. Verify important facts, decisions,
code, and advice before relying on them.

# Write Technical Documentation

Use this to create or update documentation grounded in the current system.

## Prompt

Write or update `[README, guide, reference, runbook, or other document]` for
`[software or feature]`.

Audience: `[users, contributors, operators, or maintainers]`

Reader goal: `[what they need to accomplish]`

Relevant source of truth: `[code, configuration, tests, or existing docs]`

Required topics: `[topics]`

Constraints: `[format, length, location, or terminology]`

Inspect the actual implementation before documenting it. Use task-oriented
examples, state prerequisites, explain expected results and failure cases, and
avoid claims the source does not support. Preserve useful existing context.
Verify commands and links when practical, and flag anything that remains
unconfirmed.
